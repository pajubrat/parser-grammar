['John', 'gave', 'the', 'book', 'to', 'Mary']
[[D John-]:1 [T [(DP):1 [v [gave [[the book] <to [D Mary-]>]]]]]]
'D John T __v gave the book to D Mary .'
Complexity: Failed parses 0, Merge operations required 11
Comment by authors:
The parser analyses the PP as a right-adjunct to the DP. Because right adjuncts are
virtual (but geometrical) constituents, the verb here selects for 'the book'. This is a low position
for the adverbial, but the parser finds also solutions in which it is attached to higher node. Verbal adverbials must be
positioned inside the VP or be c-commanded by the V, in current implementation. A more realistic assumption 
could be that the higher positions are preferred, or that verbal adverbial must occur inside the VP. This would be
trivial to implement, but unnecessary since nothing hinges on this when it comes to pied-piping and operator movement.

Notice that no Larsonian VP-shells are projected. This due to the fact that they involve invisible verbal heads that
turned out to be exceedingly difficult to handle when the parser can use nothing but the raw list of words as its input.
The Larsonian shells could be used in production, though. If they are, then there must exist a way of linking the 
semantic interpretation arising from the structure #34 with that resulting from the Larsonian VP-shells. Another possibility
is to try to develop the parsing algorithm in such a way that it projects the required Larsonian VP-shells, and  
also recovers from a mistake when an error is detected. It is the latter aspect that was the most problematic: how do we 
determine that a wrong invisible head has been projected instead of repoairing the problem by projecting more?

The positioning between the direct and indirect argument has been studied by using binding principles. It is important
to note here that the PP is promoted to adjunct during the derivation, and so the question of how the binding data bears on
this issue remains to be studied separately. We must first determiner at which stage of the derivation the various binding
principles apply. The question is nontrivial due to the nontrivial notion of surface-guided derivation assumed in this 
study.

Comment by authors:
The auxiliary is disambiguated lexically so that it represents C. The parser is able to handle lexical ambiguities,
but we disambiguated in the lexicon in order to avoid unnecessary derivations. In addition, the current implementation did not have
an efficient algorithm for lexical disambiguation.

The object is reconstructed into [DP V]. In the set-theoretical theory of Merge order does not matter, but we used an
left-right asymmetric merge in this study so the (i) [V DP] and (ii) [DP V] orders differ from each other. We defined the complement 
relation based on sisterhood, which means that both (i-ii) satisfy the relation. The difference between them is that (ii) satisfies also
the specifier relation. That is, in (ii) DP is both a complement and a specifier of V, whereas in (i) it is only a complement. This 
distinction is not irrelevant for semantic interpretation, because it is possible for a verb to have a lexico-semantic feature
preventing a DP-argument to occur at its SPEC position, hence ruling out (ii) while leaving option (i). These would be more agentive verbs
that those which only allow for the [V DP] order. Although we did not study this issue in detail in this study, we allowed the parser
to generate both structures. The specifier/complement relation was used to match lexical features and generate semantic interpretations
at the LF-interface.

The parser was also tested by forcing complements to be right constituents. It would then reconstruct the wh-object to Comp,VP position. 
Practical test runs showed, however, that it was not able to handle the whole corpus correctly. On the other hand, it is possible
that the problem is in some other assumption.
